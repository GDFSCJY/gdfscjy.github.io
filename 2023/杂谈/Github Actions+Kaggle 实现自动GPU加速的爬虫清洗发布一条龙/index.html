Github Actions+Kaggle 实现自动GPU加速的爬虫清洗发布一条龙

发表于 2023-08-08 分类于 杂谈 阅读次数： 本文字数： 3.3k 阅读时长 ≈ 12 分钟
沉迷搜集数据，但是又懒得手动更新，所以用GitHub Actions和Kaggle搞了个自动更新方案

虽然爬虫是一件不算困难的事情，
但是每次重复步骤：运行爬虫/数据清洗脚本，修改README，push上GitHub，发布到Kaggle。
这就有点麻烦了，需要一个“持续交付”方案来完成上述任务。

需求分析

我要完成的任务如下：

  1. 从各处搜集数据，因为生数据数量很大，且各种来源数据的处理方式都不一样，所以要分成多个并行任务
  2. 对数据进行清洗（需要GPU加速）并打包
  3. 更新到GitHub总仓库，并更新README

显然仅使用GitHub Actions不能够满足上述要求。若不使用GPU，任务超时了数据都还没清洗完。
幸好Kaggle每周都有至少30小时的GPU时间可以白嫖。

经过分析，我的任务流程如下：

flowchart LR
  START("Start")-->A["Crawl from A\nClean\nPack"]-->E["Update GitHub README\n\nPush all updates to GitHub"]-->DONE("Done")

  START-->C["......"]-->E
  START-->D["Crawl from Z\nClean\nPack"]-->E

当条件触发时：

  1. 使用Kaggle服务器从各处搜集数据、进行清洗、打包和push到GitHub总仓库
  2. 使用GitHub Actions的服务器重新统计数据并更新到GitHub仓库的README

事前准备

Kaggle账号

首先当然是注册一个Kaggle账号啦（如果没有的话）

为了在GitHub的服务器上登陆进Kaggle的服务器，需要获得Kaggle的Token

在个人Settings-API，Create New Token，获得kaggle.json，里面包含了username和key，
把Kaggle Token放到仓库的Secrets（机密）中（你也不想Kaggle账号被公开到仓库中吧）

GitHub账号

在Kaggle服务器上处理完数据后要把数据push到GitHub仓库，同样需要GitHub的Token

在个人Setting-Developer settings-Personal access tokens创建一个token，记得给repo权限

然后也存放到仓库机密中

仓库权限

GitHub Actions需要获得读写权限才能够更新仓库

在仓库的Settings-Actions-General中，Workflow permissions选择Read and write permissions

GitHub Actions提供matrix“矩阵”方法自动创建变量组合，例如下面配置

这样，GitHub Actions将创建多个并行任务，
根据include的参数，每个任务使用不同的title和code_file变量，同时其他参数保持一致

最大并行任务数设置为2，最高为3，是因为Kaggle会保留（reserve）额外任务运行时间。
Kaggle每周最少30小时运行时间（一般也是只有30小时），一个任务最大运行时间是12小时；
同时运行2个任务，Kaggle会保留12小时运行时间；运行3个，保留24小时，依次类推。
为了防止出错，就设个最大并行数吧
