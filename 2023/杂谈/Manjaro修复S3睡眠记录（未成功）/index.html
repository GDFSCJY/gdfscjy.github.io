发表于 2023-07-02 分类于 杂谈 阅读次数： 本文字数： 714 阅读时长 ≈ 3 分钟
在编译环节出现了意外的错误，在寻找解决办法

本人笔记本HP Envy X360 13-ar0000au；CPU是AMD Ryzen 3700U

在Manajaro系统使用睡眠时就会睡死，只能重新启动

于是参考了这篇博客：

https://blog.wyraz.de/linux/enabling-s3-suspend-for-linux-on-an-hp-pavilion-aero-13/

检查S3是否可用

使用dmesg检查是否支持S3：
sudo dmesg | grep "ACPI: PM: (supports"

输出：
[    0.220118] ACPI: PM: (supports S0)

是不支持S3的，所以只能使用s2idle来睡眠，而不能用更好的deep睡眠

输入：
cat /sys/power/mem_sleep

输出：
[s2idle]

工具

需要用到两个工具acpica和cpio去转储、修改和编译ACPI表，内核再加载编译后的表。

sudo pacman -S acpica cpio
