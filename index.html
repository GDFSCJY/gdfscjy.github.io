括号匹配

IDE中通常有括号匹配功能来检测语法错误。
例如有下列程序：

可以看到程序里的括号都是成双成对出现的，且含有一定的规律。
若把除了括号以外的字符去掉，则获得下列字符串：
{(){(){()}}}

这样的字符串有一个规律，从左到右扫描，后扫描到的左括号接下来会先扫描到对应的右括号。
这种规律称作“后进先出”（Last In First Out）。

栈结构能够实现这种“后进先出”的操作，也就是说可以使用栈结构来判断括号是否匹配。

括号不匹配有下列情况：

  1. 右括号缺失。例如(()。
  2. 左括号缺失。例如())。
  3. 种类不匹配。例如(]。

分析如何使用栈结构对括号字符串。以上面的字符串{(){(){()}}}为例。

  1. 扫描到第1个字符{，计算机希望和}匹配。（入栈{）
  2. 扫描到第2个字符(，计算机把}放一边，希望和)匹配。（入栈(）
  3. 扫描到第3个字符)，计算机匹配成功，希望和}匹配。（出栈(）
  4. 扫描到第4个字符{，计算机把}放在一边，希望和}匹配。（入栈{）
  5. 扫描到第5个字符(，计算机把}放在一边，希望和)匹配。（入栈(）
  6. 扫描到第6个字符)，计算机匹配成功，希望和}匹配。（出栈(）
  7. 扫描到第7个字符{，计算机把}放在一边，希望和}匹配。（入栈{）
  8. 扫描到第8个字符(，计算机把}放在一边，希望和)匹配。（入栈(）
  9. 扫描到第9个字符)，计算机匹配成功，希望和}匹配。（出栈(）
  10. 扫描到第10个字符}，计算机匹配成功，希望和}匹配。（出栈{）
  11. 扫描到第11个字符}，计算机匹配成功，希望和}匹配。（出栈{）
  12. 扫描到第12个字符}，计算机匹配成功（出栈{）
  13. 扫描结束，此时栈正好也为空。

通过分析，程序可能抛出下列错误：

  1. 程序扫描到的右括号和希望匹配到的种类不一样，例如扫描到}，希望匹配的是]，那么可以抛出种类不匹配异常。
  2. 程序扫描到右括号，栈却是空的，那么可以抛出左括号缺失异常。
  3. 程序扫描结束，栈仍有没被匹配的，那么可以抛出右括号缺失异常。

程序的执行流程如下：

    B--N-->C{"栈空？"}--N-->出栈-->D{"是否匹配？"}--Y-->A
    C--"Y(左括号缺失)"-->失败-->Z(结束)
    D--"N(种类不匹配)"-->失败
    A--N-->E{"栈空？"}--Y-->成功-->Z
    E--"N(右括号缺失)"-->失败

Optimization

PyTorch提供了许多种优化器，常用的有SGD, RmsProp和Adam等，不同的优化器算法不同，训练结果也不相同。

在[[001 PyTorch ABC#^71f1c3]]和[[001 PyTorch ABC#^6bfc3c]]中都使用了优化器对模型进行优化。

Hyperparameters

常用的超参数有epoch， batch_size, learning_rate

  • epoch：迭代整个数据集的次数。
  • batch_size：一个批次里的样本数量。
    假设batch_size为n，总样本量为k：
    flowchart LR
 subgraph 1st_batch
     1.png---2.png---etc1[...]---n.png
     end
 subgraph 2nd_batch
     n+1.png---n+2.png---etc2[...]---2n.png
     end
 subgraph etc[...]
     end
 1st_batch-->2nd_batch-->etc
  • learning_rate：每次更新模型参数的程度。太小导致模型收敛速度慢甚至不收敛；太大会导致无法收敛。

Training

一个epoch一般包含两个部分：训练和验证

  • 训练：迭代训练数据集并优化参数
  • 验证：迭代验证数据集来检测模型能力

Loss Function

损失函数用来描述预测值与真实值之间的差距，所以训练模型就是为了缩小这个差距值。
常用的损失函数有用于回归的nn.MSELoss（平均方差）和用于分类的nn.CrossEntropyLoss（交叉熵）

One-layer Model

同时有输入inputs、目标target和损失函数torch.nn.functional.binary_cross_entropy_with_logits。

Tensors & Data

Tensors

Tensors是一种特殊的数据结构，类似于数组和矩阵。
PyTorch使用Tensors来表达模型的输入、输出和其参数。
Tensors操作详情：Tensors — PyTorch Tutorials

Tensors操作类似NumPy，且可以与NumPy的数组共用内存位置。
详情：Tensors — PyTorch Tutorials

Data

PyTorch使用Dataset和DataLoader来加载数据。
Dataset存放数据和对应标签，以及即时的数据增强和数据转化等；
DataLoader从Dataset中迭代出数据以便加载到模型中。

Dataset

[[001 PyTorch ABC#^b43a6c]]的例子就是从TorchVsion库中加载MNIST数据集。
也可以从TorchAudio，TorchText中加载PyTorch预设好的音频数据集，文字数据集。

Data Loader

DataLoader用来当需要用数据时才从Dataset中调取数据并放到模型中。
根据上面自定义数据集的定义，输出的一批数据包括图像和对应标签。
[[001 PyTorch ABC#^2f269f]]的例子就是把MNIST数据集放到数据生成器中。

  1. 实例化数据集并放入Data Loader 常用的参数定义：

dataset：指定所需要加载的数据集。

batch_size：每一批次加载出多少个样本，一般图像数据集输出的图像数组形状是(C, H, W)，经过DataLoader自动组合成批次后形状变成(B, C, H, W)；
若数据集输出已经是(B, C, H, W)，不需要自动组合，则batch_size可设置为None。

shuffle：若为True，则每个epoch都将打乱顺序。

num_workers：加载数据子进程的数量，0表示仅使用主进程进行加载数据。
要在Windows中使用子进程加载数据，需要添加if __name__ == '__main__'到训练代码入口。

pin_memory：True时，加载器给出的数据是属于锁页内存的，这样数据加载到GPU速度更快，代价是占用更多的内存。

drop_last：True时，当最后数据集给出的数据样本量不足以让加载器生成一个标定量的批次时，抛弃这些样本。

prefetch_factor：提前加载prefetch_factor$\times$num_workers个批次。

persistent_workers：True时，当DataLoader迭代结束（例如一个epoch结束）时，不会被关闭，这样能加快下一次调用的速度。
