概念

Linux许可证

GNU通用公共许可证，即GPL。

Linux操作系统特点

  • 开放性
  • 高效性
  • 多用户多任务
  • 良好的用户界面
  • 设备独立性
  • 提供丰富的网络功能
  • 可靠的系统安全性
  • 良好的可移植性

Linux层次设计

用户应用程序，操作系统服务，Linux内核，硬件系统

7种运行级别 ※※

0：关机状态

1：单用户模式

2：字符界面的多用户模式（不支持网络）

3：字符界面的完整多用户模式

4：未分配使用

5：图形界面的多用户模式

6：重新启动

Linux三个主要部分及其功能

  • 内核
    是系统的心脏，是运行程序和管理硬件设备的核心程序。
  • 命令解释层
    是系统的用户界面，提供了用户与内核进行交互操作的一种接口。
  • 文件结构
     是文件存放在存储设备上的组织方法，文件结构的相互关联性使共享数据变得容易。

Shell

  • Shell作为操作系统的外壳，为用户提供了使用操作系统的接口。

  • 不同的用户登录其终端的提示符略有不同。

普通用户的提示符为$，超级用户的提示符为#

命令

bash命令一般格式

命令名 [选项] [参数1] [参数2] …

简单命令

  • pwd 查看当前工作目录
  • date 查看时间
  • who 显示目前登录系统的用户信息
  • cal 显示日历
  • uname 显示系统信息
  • wc 统计行数，单词数，字符数
  • clear 清屏

帮助命令

  • man 命令名
  • 命令名 --help
  • info 命令名
  • whatis 命令名

输入输出重定向 ※

输入重定向

把命令的标准输入重定向到指定文件

格式：
命令 < 文件名

例如 统计文件abc.txt：
wc < abc.txt

输出重定向 ※※

  • 把命令的标准输出或标准错误输出重定向到指定文件

    格式：
    命令 > 文件名

    例如 把目录内容输出到文件abc.txt：
    ls > abc.txt

  • 若要把标准输出结果追加到指定文件，使用追加重定向操作符号>>

    例如：
    ls >> abc.txt

  • 若要把错误输出到指定文件，使用2>

    例如：
    ls 2> err.file

  • 若要把标准输出和错误输出保存到指定文件，使用&>

    例如：
    ls &> output.file

管道

将一个程序或命令的输出作为另一个程序或命令的输入，管道符号|

例如 统计/usr/bin目录的字数（文件数）：
ls /usr/bin | wc -w

例如 统计abc.txt文件有多少行带”aaa”单词：
cat abc.txt | grep "aaa" | wc -l

通配符

  • ?代表该位置是一个任意的单个字符
  • *表示该位置可以是0个或多个任意字符
  • []表示括号内任何单个字符

文件目录

文件含义

  • 目录： 包含有许多文件的目录项，/表示根目录
  • 路径名 :  通过/结合在一起的一个或者多个目录及文件名的集合。
  • 相对路径:使用当前工作目录作为一个参考点目录查看文件系统时路径名称
  • 绝对路径：用根目录作为参考点

文件命名规则

  1. 不能有斜杠/和空字符\0，可以使用下划线_和句点.
  2. 区分大小写
  3. 以句点.开头的文件名是隐含文件

文件类型

普通文件：文本文件、数据文件、可执行的二进制文件

目录文件：一种特别文件，利用它可以构成文件系统的分层树状结构。

设备文件：是一种特别文件，用来标记各个设备驱动器。字符设备、块设备 。通常设备文件在/dev目录之下

符号链接文件：Linux具有为一个文件起多个名字的功能，成为链接。被链接的文件可以存放在相同的或不同的目录下。只要对一个目录下的该文件进行修改，就可以完成对所有目录下同名链接文件的修改

文件命令 ※

  • ls 文件显示命令
    • -a 显示包括隐藏文件的所有文件
    • -l 显示文件名外，还显示文件状态、权限、拥有者、文件大小等信息
  • cp 复制文件
    • cp [-fpR] [源文件或目录] [目的文件或目录]
    • -f 强制复制
    • -R 拷贝所有文件及目录
  • rm 删除文件
    • rm [-irf] [文件或目录]
    • -r 递归删除目录文件，及删除目录及其子目录
    • -f 强制删除
  • mv 移动文件（重命名）
    • mv [-if] [源文件或目录] [目的文件或目录]
    • -f 强制移动
  • wc 文件内容统计
    • wc [选项] 文件列表
    • -c 统计字节数
    • -l 统计行数
    • -w 统计字数
  • cat 查看文件信息
  • grep 匹配内容
    • grep [模式字符] [文件名1, 文件名2, …]
  • mkdir 创建目录
  • rmdir 删除目录
  • cd 改变工作目录
    • cd .. 回到上一级目录
    • cd 回到用户目录
    • cd /切换到根目录

文件权限 ※※

一个文件由1个类型和9个权限位来描述，9个权限位分为3组，每组3个权限位

- r w x r w x r w x

  • read权限：可以显示文件内容，八进制表示为4

  • write权限：可以编辑或删除文件，八进制表示为2

  • excute权限：可以执行该文件，八进制表示为1

  • 第1位是文件类型

  • 第2，3，4位是文件属主（user）的权限

  • 第5，6，7位是属主所在组（group）的权限

  • 第8，9，10位是其他用户（other）的权限

  • 以上三种人（all）

权限管理

命令 说明
chmod a+r file 所有用户对file有读的权力
chmod ug+w,o-w file 所有者及同用户组可写入file，其他人不可写入file
chmod u=rwx file 所有者权限设置为可读、可写、可执行
chmod u= file 所有人都没有权限
chmod 777 file 所有人都可读、可写、可执行
chmod 750 file 所有者可读、写、执行；同组可读，可执行；其他人没有权限
chmod -R 742 dir dir目录下所有文件权限都设置为742
chgrp group2 file 把文件所属组改成group2
chown user2 file 把文件所有人改成user2

tar打包

  • tar [-zcvf] [文件或目录]
  • 打包压缩dir目录到dir.tar.gz压缩文件：

用户和工作组

超级用户

  • root用户是超级用户，对系统任何文件都有权限，还可以管理系统
  • root的UID和GID都为0

用户管理文件

  • /etc/passwd 口令文件 用来保存用户信息
  • /etc/shadow 影子口令文件 用来保存用户密码

用户管理

  • useradd 添加用户
    • useradd user1 添加名为user1的用户
    • useradd -c ChinaCpu -d /opt/longcpu -G linuxsir,root,deinan -s /bin/tcsh longcpu
      添加用户longcpu，真实名字为ChinaCpu，家目录在/opt/longcpu，归属用户组linuxsir、root、
      beinan成员，其SHELL类型为tcsh
  • userdel 删除用户
  • passwd 设置密码
  • su 切换用户
  • usermod 修改用户属性
    • -l<帐号名称>    修改用户帐号名称。
    • -L    锁定用户密码，使密码无效。
    • -g<群组>    修改用户所属的群组。
    • -G<群组>    修改用户所属的附加群组。

工作组管理文件

  • /etc/group 组文件 用来保存组信息
  • /etc/gshadow 组影子口令文件 用来保存组密码

组管理

  • groupadd 添加组
    • groupadd group2 添加名为group2的组
    • groupadd -g 888 group2 添加名为group2的组，GID为888
  • gpasswd 为工作组设置口令
  • groupdel 删除组
  • groupmod 修改组属性
    • groupmod -n group22 group2 修改group2为group22

VI进入和退出

进入VI

在shell中输入命令vi和想要编辑（建立）的文件名，例如vi example.file

退出VI

使用转移命令完成

  • :wq 把编辑缓冲区的内容写入到文件中并退出编辑器
  • :ZZ和:x 仅当作过修改时才将缓冲区内容写到文件上
  • :q! 强行退出vi

文本编辑

插入命令

  • 在i命令之后输入的内容都插在光标位置之前，光标后的文本相应向右移动。
  • 按下Enter键，就插入新的一行或者换行。

附加命令

  • a 在光标之后插入，光标可在一行的任何位置。
  • A 在光标所在行的行尾添加文本。

打开命令

  • o 在光标所在行的下面插入一行。
  • O 光标所在行的上面插入一行。

文本删除

  • x 删除光标所在的字符
  • X 删除光标前面的那个字符
  • dd 删除光标所在的整行
  • D 从光标位置开始删除到行尾

恢复操作

  • u 如果插入后用u命令，就删除刚插入的正文；如果删除后用它，就相当于又插入刚删除的正文。
  • U 把当前行恢复成它被编辑之前的状态，不管你把光标移到该行后对它编辑了多少次。

字符检索

  • 向前检索命令的格式是： /模式〈Enter〉
  • 向后检索命令的格式是：?模式 〈Enter〉

if语句

条件

例如，测试位置参数$1是否是已存在的普通文件，以下两种方式等价

  • test -f “$1”
  • [ -f “$1” ]

条件测试

参数 功能
-f 文件名 若文件存在且为普通文件，返回真
-d 文件名 若文件存在且为目录文件，返回真
-z 字符串1 若字符串1长度为0，返回真
-n 字符串1 若字符串1长度大于0，返回真
字符串1 = 字符串2 若字符串1等于字符串2，返回真。"="也可以用"=="代替
字符串1 != 字符串2 若字符串1不等于字符串2，返回真
数1 -eq 数2 若整数数1等于数2，返回真
数1 -ne 数2 若整数数1不等于数2，返回真
数1 -lt 数2 若整数数1小于数2，返回真
数1 -le 数2 若整数数1小于等于数2，返回真
数1 -gt 数2 若整数数1大于数2，返回真
数1 -ge 数2 若整数数1大于等于数2，返回真

逻辑运算符

  1. ! 逻辑非，例如：[ ! -r $1 ]或者! test -r "$1"
  2. -a 逻辑与，例如：[  - f  "$myfile"  - a  - r  "myfile"  ]
  3. -o 逻辑或，例如：[  "$a"  -ge  0  -o  "$b"  -le  100  ]
